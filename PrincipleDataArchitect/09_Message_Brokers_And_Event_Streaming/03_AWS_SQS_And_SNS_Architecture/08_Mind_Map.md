# 🧠 Mind Map – AWS SQS & SNS Architecture

## 🗺️ AWS SQS (Simple Queue Service)
### Type
- Pull-Based Polling Message Queue.
- Point-to-Point Messaging (1 Publisher, 1 Consumer).

### Modes
- **Standard SQS:** Infinite throughput. Best-effort ordering. At-least-once delivery.
- **FIFO SQS:** Capped throughput. Exact ordering. Exactly-once processing.

### Security / Processing
- **Visibility Timeout:** Temporarily hides messages while polling. If consumer crashes, message re-appears.
- **Long Polling:** `WaitTimeSeconds=20` minimizes API requests and cuts billing drastically.
- **Dead Letter Queue (DLQ):** Prevents infinite exception loops securely automatically smartly intelligently naturally implicitly elegantly dynamically. *(Catches poison pills).*

## 🗺️ AWS SNS (Simple Notification Service)
### Type
- Push-Based Publishing Engine.
- Pub/Sub Broadcast (1 Publisher, Many Consumers).

### Components
- **Topic:** The central broadcaster.
- **Subscription:** Endpoints listening to the Topic. (HTTP, Email, SQS Queues, AWS Lambda).
- **Message Filtering:** Allows downstream consumers to specify exactly which events they want to receive (saving CPU parsing costs).
